259      Default fill strategy.

261      :param volumeLimit: The proportion of the volume that orders can take up in a bar. Must be > 0 and <= 1. 
262          If None, then volume limit is not checked. 
263      :type volumeLimit: float 
264   
265      This strategy works as follows: 
266   
267      * A :class:`pyalgotrade.broker.MarketOrder` is always filled using the open/close price. 
268      * A :class:`pyalgotrade.broker.LimitOrder` will be filled like this: 
269          * If the limit price was penetrated with the open price, then the open price is used. 
270          * If the bar includes the limit price, then the limit price is used. 
271          * Note that when buying the price is penetrated if it gets <= the limit price, and when selling the price 
272            is penetrated if it gets >= the limit price 
273      * A :class:`pyalgotrade.broker.StopOrder` will be filled like this: 
274          * If the stop price was penetrated with the open price, then the open price is used. 
275          * If the bar includes the stop price, then the stop price is used. 
276          * Note that when buying the price is penetrated if it gets >= the stop price, and when selling the price 
277            is penetrated if it gets <= the stop price 
278      * A :class:`pyalgotrade.broker.StopLimitOrder` will be filled like this: 
279          * If the stop price was penetrated with the open price, or if the bar includes the stop price, then the limit 
280            order becomes active. 
281          * If the limit order is active: 
282              * If the limit order was activated in this same bar and the limit price is penetrated as well, then the 
283                best between the stop price and the limit fill price (as described earlier) is used. 
284              * If the limit order was activated at a previous bar then the limit fill price (as described earlier) 
285                is used. 
286   
287      .. note:: 
288          * This is the default strategy used by the Broker. 
289          * It uses :class:`pyalgotrade.broker.slippage.NoSlippage` slippage model by default. 
290          * If volumeLimit is 0.25, and a certain bar's volume is 100, then no more than 25 shares can be used by all 
291            orders that get processed at that bar. 
292          * If using trade bars, then all the volume from that bar can be used.
