 66  # Note: 
 67  # Up to version 0.12 CrossAbove and CrossBelow were DataSeries. 
 68  # In version 0.13 SequenceDataSeries was refactored to support specifying a limit to the amount 
 69  # of values to hold. This was introduced mainly to reduce memory footprint. 
 70  # This change had a huge impact on the way DataSeries filters were implemented since they were 
 71  # mosly views and didn't hold any actual values. For example, a SMA(200) didn't hold any values at all 
 72  # but rather calculate those on demand by requesting 200 values from the DataSeries being wrapped. 
 73  # Now that the DataSeries being wrapped may not hold so many values, DataSeries filters were refactored 
 74  # to an event based model and they will calculate and hold resulting values as new values get added to 
 75  # the underlying DataSeries (the one being wrapped). 
 76  # Since it was too complicated to make CrossAbove and CrossBelow filters work with this new model ( 
 77  # mainly because the underlying DataSeries may not get new values added at the same time, or one after 
 78  # another) I decided to turn those into functions, cross_above and cross_below.
